### 👁️ Overview

The Model Training module takes the features generated by the Preprocessor and uses them to fine-tune the UnifiedVoice GPT acoustic model. This phase teaches the model how to map your target language's text tokens to the corresponding acoustic semantic codes, capturing the unique voice, rhythm, and emotion of your dataset.

#### 📂 1. Project & Data Discovery

Unlike other modules, the trainer relies heavily on automation to prevent configuration mismatches.

- **Select Project:** Choosing a project automatically scans its `extractions/` folder.
- **Auto-Discovery:** The system will locate your `config.yaml`, `bpe.model` (tokenizer), and the `train.jsonl` / `val.jsonl` manifests. It will refuse to start if any of these critical files are missing or mismatched.
- **Run Name:** Defines the folder name under the `trains/` directory where checkpoints and TensorBoard logs will be saved. If left blank, it defaults to the project name.

#### 🛠️ 2. Core Hyperparameters

These settings dictate how the neural network learns. 

- **Epochs:** The number of times the model will iterate over the entire training dataset.
- **Batch Size (per step):** The number of audio clips loaded onto the GPU simultaneously. Reduce this if you encounter CUDA Out-Of-Memory (OOM) errors.
- **Gradient Accumulation:** Accumulates gradients over multiple steps before updating the model's weights. *Engineering Note: True Batch Size = Batch Size × Gradient Accumulation.*
- **Learning Rate:** The step size the optimizer takes when adjusting weights. The default (`2e-5`) is highly recommended for fine-tuning.
- **Validate & Save Every:** Dictates how often (in steps) the model pauses to run the validation loop and save a `gpt_step_X.pth` checkpoint.

#### ⚙️ 3. Advanced Controls

- **Use Duration Control:** When enabled, the model explicitly learns to predict the duration of phonemes/words.
- **Duration Dropout:** Randomly drops duration information during training (Default: `0.3`). This forces the model to learn natural pacing and rhythm internally, rather than relying solely on explicit duration tags, leading to more natural-sounding inference.

#### 📦 4. Checkpoint Weight Export (Unwrapping)

During training, models are saved with metadata and optimizer states that are unnecessary for inference.

- **Unwrap and Save:** This tool strips the optimizer state and cleans the dictionary keys (e.g., removing `module.` prefixes left over from distributed training). 
- **Conformer Mapping:** If enabled, it fixes flattened nested layers (e.g., converting `.conv_pointwise_conv1` back to `.conv_module.pointwise_conv1`).
- **Result:** It takes your selected `gpt_step_X.pth` and exports a clean, lightweight `gpt.pth` file directly into your training folder, ready to be copied for inference.
